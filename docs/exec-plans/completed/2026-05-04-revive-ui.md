# Revive UI

## Goal

Add incapacitated-player UI and revive countdown rendering.

## Acceptance Criteria

- Incapacitated player sees a red death-style overlay.
- Overlay shows CKDM bold title for the actual incapacitation cause.
- Overlay shows a revive countdown sentence; only the seconds number uses CKDM bold yellow text with orange shadow.
- Overlay has a green nine-slice `GIVE UP` button that skips the revive window and lets normal death/respawn proceed.
- Active revives show `REVIVING X.XXS` above the target name tag for real players and debug dummies.

## Context Links

- [Revive docs](../../REVIVE.md)
- [Quality](../../quality.md)

## Steps

- Add revive client/server packets.
- Sync incapacitated self-state and revive progress.
- Render incapacitated screen with CKDM text and green nine-slice button.
- Render world-space revive progress above targets.
- Update docs and run checks.

## Validation

- `./gradlew build`
- `bash ./scripts/check-sonata.sh`

## Decision Log

- Button label is `GIVE UP`, because it is explicit that the player is choosing to stop waiting for revive and proceed to death/respawn.
